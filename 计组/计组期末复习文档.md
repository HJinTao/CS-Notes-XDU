## 4 存储技术

### 4.3 Cache

1. 地址映射与变换

    - 全相联
        - **地址映射表**的存储单元个数=Cache块数，每个存储单元的位数=主存块号位数+属性位数（有效位）
        - 主存地址构成：主存块号+块内地址
        - Cache地址构成：块号+块内地址
    - 直接地址映射
        - 无地址映射表
        - 主存地址构成：区号+区内块号+块内地址
        - Cache地址构成：块号+块内地址，**主存地址去掉区号就是Cache地址**
        - 注意主存不分区的第几块如何判断
    - 组相联
        - 主存地址构成：区号（标记，Tag）+组号（索引，Index）+块内地址
        - Cache地址：常问Cache路内地址，也就是Index+块内地址
        - 地址映射表：
            - 有效位+区号（标记，Tag），省略了对应该组内的Cache第几块，
            - 存储单元个数是Cache总块数
        - m路组相联，把Cache分成m路，主存根据每路Cache的大小分区
        - 如何根据主存地址查找Cache块（手写）：先根据Index去地址映射表查找，再根据地址映射表的有效位和比对地址映射表的Tag和主存地址的Tag
        - 组相联无Index位时即为全相联
        - 组相联为1路组相联时即为直接映射
    - **三种地址映射分别的特点以及适用场景**
        - **全相联**需要设置地址映射表，标记存储消耗最多，不适合大容量Cache，速度最慢，命中率最高
        - **直接映射**不需要设置地址映射表，标记存储消耗最少，速度最快，命中率最低
        - 现在计算机采用的大部分都是**组相联**

2. 替换算法

    - FIFO
    - LRU

    都是画栈图，弹栈尾，入栈顶

3. 更新策略

    关注主存与Cache的一致性问题

    - 写回法
        - Cache Hit：将数据写入Cache而不写入主存，Cache被替换出去时才写回主存
        - Cache Miss：将主存块调入Cache，再在Cache中操作
    - 写直达法
        - 在将数据写入Cache的时候同时写入主存，速度慢
    - 特性对比
        - 写回法需要设置标志位，写直达法不需要
        - 多处理器系统通常采用写直达法以保持一致性（Data race）
        - 写回法可以减少Cache与主存的通信量

4. Cache性能

    - 平均访问时间$T_A = hT_c + (1-h)(T_c+T_h)= T_c+(1-h)T_M$  
    - 加速比$S_p = \frac{T_M}{T_A}$
    - Cache性能提高
        - 多级Cache

### 4.4 虚拟存储器

1. 概念

    - 虚拟存储器有三个地址
        - 虚拟地址空间：程序员编程中用到的地址
        -  编译程序生成主存储器地址空间：实际主存的物理地址
        -  CPU地址引脚辅存地址空间：磁盘存储器的地址
    - 工作原理
        - 若地址转换硬件想要访问虚拟地址对应的数据页或者程序页
            - 若在主存中：则把虚拟地址转换为主存地址（CPU：MMU）
            - 若不在主存中：触发缺页异常，把虚拟地址转换为外存地址，并调度进入主存（OS）
    - 一些知识点：
        - 虚拟存储器包括主存和外存

    

2. 三种虚拟存储器

    - 段式虚拟存储管理（了解概念即可）
        - 将程序空间按逻辑意义分成段，按段进行调入、调出和管理。
    - 页式虚拟存储管理
        - 将主存空间和程序空间都机械等分成相同大小的页面，按页顺序编号
        - 任一主存单元的地址由实页号和页内位移两个字段组成。
        - 地址变换过程：虚页->页表 -> 实页
    - 段页式虚拟存储管理
        - 将程序按逻辑意义先分成段，再让各段和实主存都机械等分成相同大小的页面
        - 每道程序通过一个段表和相应的一组页表来进行程序在主存空间中的定位
    - 一些知识点
        - **上述三种虚拟存储管理器都是以页为方式和外存进行交换**



3. 虚拟存储器改进方案
    - 多级页表：当页表容量超过一个页面大小时，需要多级页表来管理
        - 多级页表 级数 的计算
    - 快慢表法：使虚拟存储器的速度接近主存的速度，
        - 快表（TLB）：在CPU中，高速硬件实现，小容量，类似Cache
        - 慢表：在主存中，当快表中查不到时，从主存的慢表中查找
        - 快表慢表构成一个二级存储方式，查表时，由虚页号同时查快表和慢表



4. TLB与Cache的综合计算
    - 题目中会存在三种地址：虚拟地址，物理地址（主存地址），Cache地址
        - 虚拟地址：存在两种形式
            1. 与物理地址交互时：虚拟页号+页内地址
            2. 与TLB交互时：又TLB地址变换方式决定（组相联，全相联，直接映射）
        - 物理地址（主存地址）：存在两种形式
            1. 与虚拟地址交互时：实页号+页内地址
            2. 与Cache地址交互时：由Cache地址变换的方式决定（组相联，全相联，直接映射）
    - 主存中的页表（将虚页号-> 实页号）
        - 页表项：虚页号（隐藏）+ 有效位 + 实页号



### 4.5 外部存储器

1. 磁盘存储器
    - 磁记录方式（见图）
    - 磁盘平均访问时间计算：$T=平均寻道时间＋平均等待时间＋数据传输时间$
        - 平均等待时间经常取磁头旋转半周的时间
        - 文件连续存储 vs 文件随机存储
            - 文件连续存储：平均寻道时间和平均等待时间访问一次扇区时计算一次即可
            - 文件随机存储：每次访问扇区都要计算
        - 格式化容量 VS 非格式化容量
            - 格式化容量用 每个扇区的记录容量计算
            - 非格式化容量用 位密度计算
2. 独立冗余磁盘阵列 RAID（了解即可）
    - RAID是一组物理磁盘驱动器，在操作系统下被视为一个单一的逻辑驱动器
    - 数据分布在一组物理磁盘上， 改善了I/O性能，增加了容量。
    - 冗余磁盘容量用于存储奇偶校验信息，保证磁盘万一损坏时能恢复数据。（RAID0不支持） 补偿并提高了多个磁盘的可靠性问题
3. 固态硬盘
4. 光存储盘
5. 移动存储设备



## 5 指令系统

指令系统是软件与硬件分界面的一个主要标志

### 5.1 指令集体系结构层定义

1. 存储模式

    - 数据存储顺序

        - 大端存储：最低有效字节存储在最高地址位置
        - 小端存储：最低有效字节存储在最低地址位置

        如 32位的十六进制数据12345678H，在以字节编址的主存空间4000H开始位置存储：

        |  地址  | 大端存储 | 小端存储 |
        | :----: | :------: | :------: |
        | 0x4000 |    12    |    78    |
        | 0x4001 |    34    |    56    |
        | 0x4002 |    56    |    34    |
        | 0x4003 |    78    |    12    |
        |        |          |          |

        - 注意，对于结构体而言，不同数据之间是没有大端小端顺序的，都是顺序的，也就是说大端小端是数据内的
        - 数据的起始地址指的是 最低地址，如上就是0x4000
        - 注意，画地址的时候都是从小地址到大地址，在下面画堆栈的时候也一样

    

    - 边界对齐：某数据的起始地址应该是该数据类型大小的整数倍。

        - 按字节编址的情况下

        - 16位字长的数据：起始地址为2的整数倍。

        - 32位字长的数据：起始地址为4的整数倍。

        - 64位字长的数据：起始地址为8的整数倍。

    

    - 堆栈（Stack）
        - 三个地址寄存器
            1. 堆栈指针：Stack Pointer ，SP
            2. 堆栈基址：Stack Base，SB（栈底）
            3. 堆栈界限：Stack Limit，SL（栈顶）
        - 堆栈是从大地址往小地址增长，也就是说SB地址大于SL地址，画图的时候还是遵循从小地址到大地址画图

    

    - 冯诺伊曼结构 VS 哈佛结构
        - 冯诺依曼结构：指令与数据，程序区与数据区混合，Intel，AMD采用
            - 充分利用主存资源
        - 哈佛结构：指令与数据，程序区与数据区分开，RISC采用
            - 指令存储器只读，设计、控制简单；
            - 避免了数据对程序可能造成的破坏；
            - 利用n位地址可以获得2个2n大小的地址空间。

2. I/O 模式
    - 统一编址：大多数RISC处理器，采用统一的输入输出指令
    - 独立编址：Intel/AMD x86处理器，有独立的输入输出指令





### 5.2 寻址方式

- 指令获取操作数的方式，一般左侧都是目的操作数，右侧是源操作数。
- 常见寻址方式

```assembly
MUL BL           隐含寻址
MOV R1, #1200H   立即寻址
MOV R1, RB       寄存器寻址
MOV R1, (1200H)  直接寻址
MOV R1, (X)      间接寻址
MOV R1, X 		 直接寻址
MOV R1, (RB)     寄存器间接寻址
MOV R1, 1100H(RB) 寄存器基址寻址
MOV R1, (RB)(RI) 
MOV R1, 1100H(RB)(RI)  变址寻址

```

- 相对寻址

    - 注意PC相对寻址 PC方式的PC位置，PC一般是指向当前执行指令的下一条指令位置
    - 地址偏移量是补码编码

- 注意当寄存器大小与字长不一致时的读取方式，与大端，小端有关，见附件题

- 区分汇编命令中的()和说明中的()

    - 主存地址

        ```assembly
        MOV R1, (1100H)  #表示将Memory(1100H) -> R1  立即数
        MOV R1, (X)      #表示将Memory(Memory(X)) -> R1
        MOV R1, X        #表示将Memory(X) -> R1
        
        #特殊
        CALL X       	 #表示将 X -> PC
        CALL (X)     	 #表示将 Memory(X) -> PC
        ```
    
    
    
    - 寄存器
    
        ```assembly
        MOV R1,  R2    # 表示将(R2) -> R1,也就是将R2的内容传递给R1
        MOV R1, (R2)   # 表示将Memory((R2)) -> R1
        ```

## 6 中央处理器

### 6.1 概述

1. CPU的组成

    - DP：数据通路，存储单元/寄存器组 <-> ALU

    - CU：控制单元（根据不同CU的实现方式分为  **硬布线控制器** 和 **微程序控制器**）

2. 指令周期，CPU周期，节拍周期

    指令周期：CPU执行一条指令的时间

    - （CPU周期）取指令子周期
        - （节拍周期）微操作
        - （节拍周期）微操作
    - （CPU周期）执行指令子周期
        - （节拍周期）微操作
        - ……
    - （CPU周期）中断子周期

3. 微操作 VS 微命令

    - 微操作：$AR\gets PC$，CPU可以实现的、不可分解的操作动作
    - 微命令：$AR_{in},PC_{out}$，控制微操作完成的控制信号称为微命令

4. 各种CPU周期的微操作序列

    - 取指子周期
    - 中断子周期
    - 执行子周期
        - 注意CALL命令 与 SUB命令的间接寻址 微操作序列
        - 注意PUSH和POP R0命令，先写出逻辑等价式再写微操作序列会比较好理解



### 6.2 硬布线控制器设计

1. **各种微操作-微命令序列**
2. 注意点
    - DRI：DR对 CPU内部总线的信号
    - DRS：DR对CPU外部总线的信号
    - 没有$Z_{in}$和$Y_{out}$
    - 注意微命令序列顺序不要写错



### 6.3 微程序控制器设计

1. 微指令，微程序
    - 微指令：对在一个时间单位（节拍）内出现的一组微操作进行描述的语句称作微指令
    - 微程序：一个微指令序列称作微程序，一般一条机器指令对应一条微程序
2. Control Memory ，CM，控制存储器
    - CM在 CPU内，ROM
    - 控制存储器中存放着指令系统中定义的所有机器指令的微程序
    - 容量计算：微指令数(存储单元个数) * 微指令位长
3. 微指令设计
    - 微指令格式：地址域（指向下一条微指令地址） + 控制域（产生控制信号）
    - 根据地址域的不同分为
        1. 两地址格式（断定方式）
            - 微指令格式：AC（地址选择位） + 地址1 + 地址 2 + 控制域
            - AC选择地址1 or 地址2，一般是顺序地址和跳转地址
            - $\mu AR$ 和 $\mu IR$
        2. 单地址格式（计数方式，增量方式）
            - 微指令格式：AC + 地址 + 控制域
            - AC 使能地址，若无效则顺序地址，$ \mu PC + 1 -> \mu PC$
            - $\mu PC 和 \mu IR$
        3. 可变格式
            - 存在两只微指令格式：控制微指令和跳转微指令
            - 控制微指令：标识 S + 控制域
            - 跳转微指令：标识 S + 分支控制 + 跳转地址
            - 微指令的位长取 两种微指令字长的最大值
    - 根据控制域的不同分为
        1. 垂直型微指令控制域编码（了解即可）
        2. 水平型微指令控制域的编码
            - 直接表示法：一个控制信号对应一个控制域位
            - 译码法：n位长度的控制域表示$2^n$个控制信号
            - 字段译码法：把控制域分为若干字段
                - 相容信号和互斥信号：能同时生效的控制信号就是相容信号，反之互斥
                - 字段间控制信号相容，字段内控制信号互斥
                - 注意计算每个字段的最少位数时记得加上 NOP后再计算，如4个互斥微命令需要$log(4+1）=3$位编码



### 6.4 微程序控制器 VS 硬布线控制器

- 微程序控制器
    - 比硬布线控制器速度慢
    - 设计简单化、规范化
    - 功能可修改、可扩充
    - 实现成本低，出错概率小
    - 常用于CISC处理器控制器的实现
    - 控制存储器容量＝微指令长度×平均每指令微程序长度(微指令数)×指令数
- 硬布线控制器
    - 速度快
    - 当计算机系统复杂时，设计困难
    - 一旦实现，不可修改和扩充
    - 常用于RISC处理器控制器的实现

    - CPI与IPC互为倒数

    - MIPS只说明了指令执行速度，不能反应真实性能，性能还是要以CPU时间为准
    - 指令集不同的计算机 MIPS无法比较

4. Flops
    - CPU每秒完成的浮点运算次数。





## 7 流水线技术



### 7.1 概述

1. 流水线不能缩短单个任务的响应时间，但可以提高吞吐率
2. 流水线类型
    - 按位于计算机系统层次划分
    - **按流水线功能划分**
        - 单功能流水线
        - 多功能流水线
            - 静态流水线
            - 动态流水线
    - 按流水线是否有反馈回路划分
    - 按任务顺序划分
    - 按处理数量划分



### 7.2流水线性能测量

1. 时空图：空间（段）-时间（时间周期）

2. 总完成时间

    - $m级流水线 完成n个任务所需要的时间： T_n(m)$（第一个任务完成的时间 + 剩下的任务完成需要的时间）
        - 流水线各段时间相等：$T_n(m) = [m + (n - 1)] \cdot \tau$
        - 流水线各段时间不等：$T_n(m) = \sum_{i=1}^{m}  \tau_i + (n - 1) \cdot \max\{ \tau_i \}$

    - $非流水线处理时间T_n(1)=n\cdot m \cdot \tau$

3. $吞吐率TP$

    - 最大吞吐率$TP = \frac{1}{max\{\tau_i \}}$

    - 实际吞吐率$TP = \frac{n}{T_n(m)}$

4. **加速比S**

    - $若没给具体任务数n，则按n=1，S = \frac{T(1)}{T(m)}$

    - $若给了具体任务数则按S = \frac{T_n(1)}{T_n(m)}$

5. **效率E**

    - 时空图的面积占比即为效率

    - 流水线各段时间相等：$E=\frac{T_n(1)}{m \cdot  T_n(m)}=\frac{n\cdot \tau}{T_n(m)}$
    - 流水线各段时间不等：$E=\frac{T_n(1)}{m \cdot  T_n(m)}$

6. **填充时间** = $(m-1) \cdot \tau$ （流水线各段时间相等）

7. **流水线气泡**

    - 以书中例题为例，若流水线出现断流，则总时间需要包括重新填充时间
    - 注意重新填充时间和跳过的指令数无关！一定是上述的填充时间公式

8. 综合大题

    - 多功能流水线
        - 动态流水线：各段可以同一时间执行不同的功能
        - 静态流水线：各段同一时间只能执行相同的功能
        - 注意点
            1. 先列出指令顺序再进行时空图绘制
            2. 以向量点积运算流水线为例，先算乘法指令再算加法指令，注意加法运算符合结合律可以简化
    - 各段时间不等流水线：
        - 选取时间最长的那段流水线作为间隔时间
